# Governance

## Project Leadership

**Project Maintainer**: ShunsukeHayashi

The project maintainer is responsible for:
* Reviewing and merging pull requests
* Release management
* Setting technical direction
* Ensuring the project follows best practices

## Decision Making

### Technical Decisions

Technical decisions are made by the project maintainer with input from the community. Major changes will be discussed in Issues or Pull Requests before implementation.

### Non-Technical Decisions

Non-technical decisions (e.g., governance changes, budget) are made by the project maintainer.

## Contribution Recognition

All contributors are recognized in:
* CONTRIBUTORS.md file
* Release notes for significant contributions
* GitHub contributor statistics

## Conflict Resolution

1. **Discussion**: Discuss the issue in the relevant Issue or PR
2. **Escalation**: If unresolved, tag the maintainer for review
3. **Final Decision**: The maintainer makes the final decision

## Governance Changes

Changes to this governance model require:
* Proposal via Issue or PR
* Community discussion
* Approval by project maintainer

## Version Control

The project follows [Semantic Versioning](https://semver.org/):
* **MAJOR** version for incompatible API changes
* **MINOR** version for backwards-compatible functionality
* **PATCH** version for backwards-compatible bug fixes
